# Publicacao Gratuita com GitHub Releases

## Quando usar

Use este caminho quando voce quer publicar o APK gratuitamente sem entrar em loja paga.

## O que publicar

- `downloads/fast-crochet-android14-debug.apk`
- opcionalmente um `.zip` com codigo-fonte e documentacao

## Passo a passo

1. Crie um repositorio no GitHub.
2. Envie o codigo do projeto.
3. Abra a aba `Releases`.
4. Crie uma nova release com tag, por exemplo `v1.0.0`.
5. Anexe o arquivo APK.
6. Publique a release.
7. Compartilhe o link da release ou o link direto do asset.

## Recomendacao de nomenclatura

- Tag: `v1.0.0`
- Titulo: `Fast Crochet Android 14+ v1.0.0`
- Asset:
  - `fast-crochet-android14-debug.apk`
  - `fast-crochet-source-v1.0.0.zip`

## Vantagens

- custo zero
- entrega imediata
- facil de atualizar por versao
- bom para MVP, beta fechado e validacao comercial

## Cuidados

- o usuario precisa instalar por sideload
- para distribuicao publica ampla, prefira depois uma loja oficial
- troque o APK de debug por um APK release assinado antes da divulgacao ampla

## Materiais que devem acompanhar a release

- instrucoes de instalacao
- changelog curto
- politica de privacidade
- termos de uso
- e-mail de suporte
